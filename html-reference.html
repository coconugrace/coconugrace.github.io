HTML

The HTML tags are used at the very beginning and end of your code. It signals the computer that your code is written in HTML.

Syntax

<html>All your html code belongs here</html>

Example

Input

<html>
<body>
<p>Hello world!</p>
</body>
</html>

Output

Hello world!




Head

The head is located before the body, containing metadata (data about the page). Whatever is put in the head does not actually appear on the page. The head includes things like the title or style of the page.

Syntax

<head>Metadata about the page</head>

Example

Input

<head>
<style>
body {background-color: teal;} </style>
</head>




Title

The title is found in the head. It determines the title of the page (which the user can see on the tab at the top), as well as in the results of a search engine (such as Google) and in favorites/bookmarks. Again, the title cannot be seen on the page itself.

Syntax

<title>Title of page</title>

Example

Input

<head>
<title>How to HTML</title>
</head>




Body

The body is the place where the most of you content is placed- including headings, paragraphs, images, tables, etc.

Syntax

<body>All the content in the page</body>

Example

Input

<body>
<p>Hello World!</p>
</body>

Output

Hello World!




Headings

Headings are used to signify importance over other content. The heading tags go around the words you want to enlargen. Headings are bigger and bolder. There are six types of headings- h1 being the biggest,, and h6 being the smallest.

Syntax

<h4>This would be a size 4 heading</h4>

Example

Input

<h1>All about HTML</h1>

Output

All about HTML




Paragraphs

When you have a great amount of writing to include, you probably want to make it a paragraph. Paragraphs are surrounded by its paragraph tags and are often styled.

Syntax

<p>This is your paragraph</p>

Example

Input

<p>You first need a computer to write HTML.</p>

Output

You first need a computer to write HTML.




Breaks

Breaks create spaces between lines. For example, if you want to write an acrostic poem, you don't want all the letters to be squished up on one line. Breaks make adding a new line possible. Unlike paragraphs, breaks don't need an end tag.

Syntax

<br>

Example

Input

Coding<br>is<br>hard.

Output

Coding
is
hard.




Lines

This element is similar to a break, because is indicates a significant pause in the text. The only different is that a line runs across the page to separate the text.

Syntax

<hr>

Example

Input

Now that you have gathered the materials needed to learn HTML, let us proceed. <hr>First, open up your computer.

Output

Now that you have gathered the materials needed to learn HTML, let us proceed. First, open up your computer.


Comments

People often use comments as personal note in their code. Comments cannot be seen on the page, but only in the code. Coders also comments to hash code out in order to debug their program.

Syntax

<!-- This is your comment -->

Example

Input

Chromebooks suck.<!-- Check credibility of this statement. -->

Output

Chromebooks suck.




Links

When coders want to redirect the user to a new place, they often use hyperlinks to do so. Hyperlinks can be placed on text or images, that when clicked, redirects to a new place on the web.

Syntax

<a href="url you want to go to">the text that appears</a>

Example

Input

<a href="https://en.wikipedia.org/wiki/HTML">Learn more about HTML!</a>

Output

Learn more about HTML!


Images

To insert images into your website, you must paste the image link into your code. You can resize the image by setting the height and width of it within the image tag. Images also usually have alternative text that appears when the image cannot be retrieved.

Syntax

<img src="image url" alt="alternative text">

Example

Input

<img src="https://upload.wikimedia.org/wikipedia/commons/6/62/Microsoft-Sharp-Kin-Two-Phone.jpg" alt="Phone">

Output




Lists

Lists are used to add bullet points or number your text. There are many different types of lists, including different fonts of unordered lists, as well as lists numbered by lowercase Roman numerals, uppercase Roman numerals, and Arabic numbers. Lists can be as long as you want (of course don't try to crash your computer though).

Syntax

Ordered list:

<ol>
<li>content of bullet 1</li>
<li>content of bullet 2</li>
<li>content of bullet 3</li>
</ol>

Unordered list:

<ul>
<li>content of bullet 1</li>
<li>content of bullet 2</li>
<li>content of bullet 3</li>
</ul>

Example

Input

<p>Grocery list:</p>
<ol>
<li>Cookies</li>
<li>Chocolate</li>
<li>Crackers</li>
</ol>

Output

Grocery list:

  1. Cookies
  2. Chocolate
  3. Crackers



Tables

Inserting tables are generally complicated and require more work. Every table has its start and end tag, as well as tags indicating each new row (tr), table header (th), and table data (td).

Syntax

<table>
<tr>
<th>Header content of 1st data cell in row 1</th>
<th>Header content of 2nd data cell in row 1</th>
<th>Header content of 3rd data cell in row 1</th>
</tr>
<tr>
<td>Content of 1st data cell in row 2</td>
<td>Content of 2nd data cell in row 2</td>
<td>Content of 3rd data cell in row 2</td>
</tr>
</table>

Example

Input

<table> <tr>
<th>Snickerdoodles</th>
<th>Chocolate chip</th>
<th>Gingerbread</th>
</tr>
<tr>
<td>72/td>
<td>89</td>
<td>0/td>
</tr> </table>

Output

Snickerdoodles Chocolate chip Gingerbread
72 89 0



Blocks

This element is used to divide content up and separate it into its own line (similar to using breaks). It can also be used to set aside a certain text for styling or other elements. The opposite of this would be inline elements, which appear on the same line as adjacent content.

Syntax

<div>Content of 1st section</div>

Example

Input

<div style="color:teal;">
<h6>Hello world!</h6>
<h5>This is some sick HTML</h5>
</div>

Output

Hello world!
This is some sick HTML
